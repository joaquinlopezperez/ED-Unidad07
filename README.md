ACTIVIDAD 1

El diagrama de casos de uso representa las interacciones entre los actores Socio y Administrador con el sistema del centro deportivo. Se identifican funciones como reservar pista o gestionar pagos. Se utiliza «include» para la verificación obligatoria de pagos y «extend» para el seguro opcional en clases. El diagrama de actividades muestra el flujo de reserva, incluyendo decisiones sobre disponibilidad y estado de pagos.

ACTIVIDAD 2

El diagrama de actividades describe el proceso de compra de billetes, diferenciando el flujo según el método de pago (tarjeta o efectivo). El diagrama de estados representa el ciclo de vida del billete desde su emisión hasta su uso o caducidad. Se incluyen transiciones provocadas por eventos como la compra, validación en el torno o expiración del billete.
